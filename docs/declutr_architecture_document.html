Declutr

Your personal knowledge operating system — private, AI-powered, and built for the way you actually think.

Declutr stores every document, note, and file in an end-to-end encrypted Vault, then gives you an AI copilot that can search, summarize, and reason over everything you've ever saved — organized around the Life Areas, Projects, and Goals in your life.

🔒 Zero-Knowledge Vault
SRP-based auth, client-side encryption. Server never sees your data.
🤖 Grounded AI Copilot
RAG-powered chat with citations from your own files.
🧠 Semantic Search
Hybrid keyword + vector search across all assets.
🗂️ Life OS
Life Areas → Projects → Goals instead of folders.
🔮 Predictive Intelligence
Proactive suggestions before you ask.
🤝 Multi-Agent Platform
Specialist AI agents that collaborate on complex tasks.
🏢 Organizations
Multi-tenant workspaces with role-based access control.
🧩 Extension Marketplace
Sandboxed, versioned third-party extensions.

Authentication

Declutr uses SRP (Secure Remote Password) — a zero-knowledge protocol. The server never sees your password, not even at registration.

Registration Flow

Client generates SRP verifier from password
→
POST /api/v1/auth/register (email + verifier + salt)
→
Server stores verifier (never the password)

Login Flow

POST /auth/login/start
→
Server sends SRP challenge
→
Client computes proof
→
POST /auth/login/finish
→
Server verifies proof, returns JWT

API Endpoints

MethodEndpointDescription
POST/api/v1/auth/registerCreate account (SRP verifier + salt)
POST/api/v1/auth/login/startBegin SRP handshake
POST/api/v1/auth/login/finishComplete SRP handshake, get JWT
GET/api/v1/meGet authenticated user (requires JWT)

Upload Pipeline

Client selects file
→
POST /api/v1/vaults/:id/assets
→
Store to R2
→
Queue processing job
→
OCR / text extraction
→
Generate embeddings
→
Index in search

The processing queue is handled by background workers managed by the supervisor package. Workers run as goroutines inside the server process.

Search Pipeline

Declutr uses a hybrid search approach — combining full-text BM25 keyword search with vector semantic search, then fusing results with Reciprocal Rank Fusion (RRF).

Query string
→
Parallel: BM25 + Vector similarity
→
RRF score fusion
→
Ranked results + snippets

Deployment

Declutr is designed for simple, affordable cloud deployment. No Kubernetes. No Helm. No Terraform.

Recommended Stack

Backend
Railway · Render · Fly.io
Frontend
Vercel · Netlify
Database
Supabase · Neon (Postgres)
Storage
Cloudflare R2
Cache
Upstash Redis

Deploy Backend to Railway

railway login
railway link
railway up

Deploy Frontend to Vercel

cd frontend
vercel deploy --prod

Local Development (Docker)

docker compose up   # Starts Postgres + Redis + Backend + Frontend

Quick Start

git clone https://github.com/lakshhchopra/declutr.git
cd declutr

# 1. Set up environment
cp .env.example .env
# Edit .env with your API keys

# 2. Start database + cache
docker compose up -d db redis

# 3. Run backend
cd backend
go run cmd/server/main.go

# 4. Run frontend (new terminal)
cd frontend
npm install
npm run dev

# App → http://localhost:3000
# API → http://localhost:8080
